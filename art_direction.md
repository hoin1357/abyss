# 픽셀 그래픽 리소스 가이드라인

## 아트 스타일

'심연의 탐험가'는 클래식한 로그라이크 게임의 느낌을 유지하면서도 현대적인 웹 게임에 맞는 선명하고 가독성 높은 픽셀 아트 스타일을 추구합니다. DCSS(Dungeon Crawl Stone Soup)와 Shattered Pixel Dungeon의 그래픽 스타일을 참고하되, 완전히 동일하지 않도록 독창적인 요소를 가미합니다.

### 주요 아트 방향성
- **간결하고 명확한 디자인**: 작은 해상도에서도 각 요소가 명확히 구분되도록 합니다.
- **층별 고유 분위기**: 각 층마다 색상 팔레트와 시각적 요소를 차별화하여 독특한 분위기를 조성합니다.
- **일관된 스타일**: 캐릭터, 타일, UI 요소 모두 일관된 스타일로 제작하여 시각적 통일성을 유지합니다.
- **동적인 요소**: 간단한 애니메이션을 통해 게임에 생동감을 부여합니다.

## 기술적 사양

### 해상도 및 그리드
- **타일 크기**: 16x16 픽셀 (기본 단위)
- **캐릭터 크기**: 16x16 픽셀 (기본), 보스는 32x32 또는 48x48 픽셀
- **UI 요소**: 8의 배수로 크기 설정 (8x8, 16x16, 24x24 등)
- **그리드 시스템**: 모든 게임 요소는 16x16 그리드에 맞춰 배치

### 컬러 팔레트

게임 전체적으로 제한된 컬러 팔레트를 사용하여 일관된 시각적 스타일을 유지합니다. 각 층별로 기본 팔레트에 특화된 색상을 추가합니다.

#### 기본 팔레트
- **어두운 색상**: #1a1a2e, #16213e, #0f3460
- **중간 색상**: #533483, #5b8fb9, #7b2cbf
- **밝은 색상**: #e94560, #fcf8e8, #94b49f
- **강조 색상**: #eca869, #61764b, #9c9ef3

#### 층별 특화 팔레트
1. **하수도 (1-3층)**
   - 주요 색상: 녹색, 갈색, 회색
   - 강조 색상: 독성 녹색(#39ff14), 녹슨 갈색(#8b4513)

2. **광산 (3-6층)**
   - 주요 색상: 갈색, 회색, 어두운 파랑
   - 강조 색상: 광석 파랑(#4169e1), 황금색(#ffd700)

3. **신비한 심해 (7-9층)**
   - 주요 색상: 파랑, 청록색, 보라색
   - 강조 색상: 발광 청록색(#00ffff), 심해 파랑(#000080)

4. **최종 심연 (10층)**
   - 주요 색상: 검정, 보라, 빨강
   - 강조 색상: 마법 보라(#9370db), 불길한 빨강(#8b0000)

## 그래픽 에셋 목록

### 캐릭터
1. **플레이어 캐릭터**
   - 기본 모습 (4방향)
   - 공격 모션 (4방향)
   - 피해 모션
   - 사망 모션

2. **적 캐릭터**
   - **하수도**: 거대 쥐, 슬라임, 하수도 악어
   - **광산**: 광산 골렘, 박쥐 무리, 광부 유령
   - **심해**: 심해어, 수정 골렘, 고대 수호자
   - **보스**: 광산 드릴러, 심해의 사제, 거대 뱀

3. **NPC**
   - 하수도 관리인
   - 잃어버린 광부
   - 고대 학자의 영혼

### 환경 타일
1. **하수도**
   - 바닥 타일 (여러 변형)
   - 벽 타일 (여러 변형)
   - 물 타일 (애니메이션)
   - 파이프, 사다리, 문

2. **광산**
   - 바닥 타일 (여러 변형)
   - 벽 타일 (여러 변형)
   - 광석 타일
   - 지지대, 광산 카트, 도구

3. **심해**
   - 바닥 타일 (여러 변형)
   - 벽 타일 (여러 변형)
   - 물 타일 (애니메이션)
   - 수정, 고대 기둥, 제단

4. **최종 심연**
   - 바닥 타일 (여러 변형)
   - 벽 타일 (여러 변형)
   - 신비한 문양 타일
   - 제단, 봉인석

### 아이템
1. **무기**
   - 근접 무기 (검, 도끼, 둔기 등)
   - 원거리 무기 (활, 투척 무기 등)
   - 마법 무기 (지팡이, 오브 등)

2. **방어구**
   - 갑옷, 투구, 장갑, 신발

3. **소비 아이템**
   - 포션, 스크롤, 음식

4. **특수 아이템**
   - 퀘스트 아이템
   - 유물

### UI 요소
1. **HUD**
   - 체력/마나 바
   - 경험치 바
   - 미니맵
   - 상태 아이콘

2. **메뉴**
   - 인벤토리 배경
   - 스킬 트리 배경
   - 버튼 (여러 상태)
   - 창 프레임

3. **전투 UI**
   - 공격 범위 표시
   - 데미지 숫자
   - 효과 아이콘

## 애니메이션 가이드라인

### 캐릭터 애니메이션
- **이동**: 2-4프레임 걷기 애니메이션
- **공격**: 2-3프레임 공격 모션
- **피해**: 1-2프레임 피해 모션
- **사망**: 3-4프레임 사망 모션

### 환경 애니메이션
- **물**: 2-3프레임 물결 애니메이션
- **불/마법**: 3-4프레임 효과 애니메이션
- **함정**: 2-3프레임 작동 애니메이션

### UI 애니메이션
- **버튼**: 호버/클릭 상태 변화
- **알림**: 페이드 인/아웃 효과
- **전투 효과**: 2-3프레임 효과 애니메이션

## 참고 게임 그래픽 변형 원칙

DCSS와 Shattered Pixel Dungeon의 그래픽을 참고할 때 다음 원칙을 적용하여 독창성을 유지합니다:

1. **색상 변경**: 기존 에셋의 색상 팔레트를 우리 게임의 팔레트로 변경
2. **비율 조정**: 캐릭터와 오브젝트의 비율을 약간 수정
3. **디테일 추가/제거**: 기존 에셋에 새로운 디테일을 추가하거나 일부 제거
4. **스타일 통일**: 여러 소스에서 가져온 에셋들의 스타일을 통일
5. **애니메이션 수정**: 기존 애니메이션 프레임을 수정하거나 프레임 수 변경

## 에셋 제작 워크플로우

1. **레퍼런스 수집**: DCSS, Shattered Pixel Dungeon 등의 참고 자료 수집
2. **스케치**: 기본 형태와 색상 배치 스케치
3. **픽셀 작업**: 16x16 그리드에 픽셀 단위로 작업
4. **색상 적용**: 정의된 팔레트 내에서 색상 적용
5. **애니메이션**: 필요한 경우 프레임별 애니메이션 작업
6. **최적화**: 웹 게임에 적합하도록 파일 크기 최적화

## 파일 포맷 및 구조

- **스프라이트시트**: PNG 포맷, 투명 배경
- **애니메이션**: 스프라이트시트 또는 개별 PNG 파일
- **파일 구조**:
  ```
  assets/
  ├── characters/
  │   ├── player/
  │   ├── enemies/
  │   └── npcs/
  ├── environment/
  │   ├── sewers/
  │   ├── mines/
  │   ├── abyss/
  │   └── final/
  ├── items/
  └── ui/
  ```
